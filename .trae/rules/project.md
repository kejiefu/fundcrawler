## 项目规则

### 股票代码格式约定

#### 1. 代码格式规范

| 表名 | code 字段格式 | 说明 | 示例 |
|------|--------------|------|------|
| `a_share_stock_basic` | 带市场前缀 | 存储股票基本信息 | `sh600519`, `sz000858`, `bj831010` |
| `a_share_kline` | 纯数字格式 | 存储K线数据 | `600519`, `000858`, `831010` |
| `a_share_indicator` | 纯数字格式 | 存储技术指标数据 | `600519`, `000858`, `831010` |

#### 2. 市场前缀说明

- `sh` = 沪市（上海证券交易所）
- `sz` = 深市（深圳证券交易所）
- `bj` = 北交所（北京证券交易所）

#### 3. 字段注释规范

在定义包含股票代码的表或字段时，**必须在注释中明确说明代码格式**：

```python
# 正确示例 - 带前缀
code = Column(String(10), comment="证券代码（带市场前缀：sh=沪市, sz=深市, bj=北交所，如sh600519）")

# 正确示例 - 纯数字
code = Column(String(10), comment="证券代码（纯数字格式，不带市场前缀，如600519）")
```

#### 4. API 接口规范

- **输入参数**：API 应同时支持带前缀和不带前缀的代码格式
- **输出数据**：保持与数据库存储格式一致
- **转换函数**：使用 `normalize_stock_code()` 和 `extract_pure_code()` 进行格式转换

#### 5. 代码转换工具

```python
def normalize_stock_code(code: str) -> str:
    """将纯数字代码转换为带前缀格式"""
    # 6xxx, 9xxx -> sh
    # 8xxx, 4xxx -> bj  
    # 其他 -> sz

def extract_pure_code(code: str) -> str:
    """从带前缀代码中提取纯数字"""
    # sh600519 -> 600519
    # sz000858 -> 000858
```

#### 6. 注意事项

- 跨表关联时，注意代码格式的一致性
- 查询 K 线数据时，确保使用纯数字格式
- 查询股票基本信息时，确保使用带前缀格式
- 前端传递代码时，注意格式转换

#### 7. 数据同步规范

- 从外部数据源获取数据时，统一转换为规范格式
- 存储前验证代码格式正确性
- 日志记录时保留原始格式便于排查问题
